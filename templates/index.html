Pre-processing Pipeline

  1. Generation: 3,000 samples created with structured probabilistic sampling; risk score calibrated to match published STATS19 2022 class proportions.
  2. Train/Test Split: 80/20 split using train_test_split with stratify=y to preserve class ratios in both sets.
  3. Feature Scaling: StandardScaler (z-score normalisation: z = (x − μ) / σ) applied to SVM and Neural Network inputs. Fitted on training data only; applied to test data to prevent data leakage.
  4. Class Imbalance: Handled via class_weight='balanced' in RF and SVM; weights inversely proportional to class frequency.
01

Random Forest Classifier

Ensemble Learning — Bootstrap Aggregating (Bagging)

How It Works

Random Forest builds 200 independent Decision Trees (CART), each trained on a random bootstrap sample of the training data (sampling with replacement). At every split, only √p features are considered (where p = number of features = 9, so √9 = 3 features per split). This decorrelates trees and reduces variance. Final prediction is the majority vote across all 200 trees.

Mathematical Basis

Gini Impurity at node t:

G(t) = 1 − Σₖ p(k|t)²

Information Gain (split criterion):

IG = G(parent) − Σ (|child| / |parent|) × G(child)

Ensemble prediction (majority vote):

ŷ = mode { h₁(x), h₂(x), … , h₂₀₀(x) }

02

Support Vector Machine (SVM)

Kernel-based Discriminative Classifier — RBF Kernel | One-vs-One Multi-class

How It Works

SVM finds the maximum-margin hyperplane separating classes in a transformed feature space defined by the kernel function. The RBF (Radial Basis Function) kernel maps data to an infinite-dimensional space, allowing non-linear decision boundaries. For 3 classes, One-vs-One (OvO) strategy trains 3 binary SVMs (Slight vs Serious, Slight vs Fatal, Serious vs Fatal); final label determined by majority vote. Probabilities are obtained via Platt scaling.

Mathematical Basis

RBF Kernel:

k(xᵢ, xⱼ) = exp(−γ ‖xᵢ − xⱼ‖²)

Primal optimisation (soft-margin):

min ½‖w‖² + C Σᵢ ξᵢ   s.t. yᵢ(wᵀxᵢ + b) ≥ 1 − ξᵢ

Decision function:

f(x) = sign( Σᵢ αᵢ yᵢ k(xᵢ, x) + b )

Hyperparameters Used

Why Suitable

  • Strong theoretical generalisation bounds (VC dimension)
  • RBF kernel models complex feature interactions (e.g. speed × alcohol)
  • Effective in moderate-dimensional spaces (~10 features)
  • ⚠️ Requires StandardScaler pre-processing
03

Multilayer Perceptron (Neural Network)

Feedforward ANN — Architecture: Input(9) → 128 → 64 → 32 → Output(3)

How It Works

The MLP consists of an input layer (9 neurons), three hidden layers (128 → 64 → 32 neurons with ReLU activation), and an output layer (3 neurons, Softmax). Training uses Backpropagation — the gradient of the cross-entropy loss is propagated backwards using the chain rule to update weights via the Adam optimiser. Early stopping monitors validation loss and halts training when it stops improving (avoids overfitting).

Mathematical Basis

Neuron computation:

z = Σᵢ (wᵢ xᵢ) + b  ;  a = ReLU(z) = max(0, z)

Softmax output (probabilities):

P(class k) = exp(zₖ) / Σⱼ exp(zⱼ)

Loss function (cross-entropy):

L = − Σₖ yₖ log(P(class k))

Adam weight update:

w ← w − α × m̂ / (√v̂ + ε)   (adaptive learning rate)
